Auto Save

This tutorial introduces an Auto Save feature which allows to automatically and periodically save a project to a separate file, so that if something happens to the file, the user may revert to that previously saved state without losing too much work.

In the previous couple of tutorials we looked at Application and Document events. By combining some of those events and auto run procedures introduced in Tutorial: Introduction to Auto Run Procedures, we can create our own auto save tool. For example, suppose:

  • We want ESPRIT EDGE to automatically save a copy of our file after we have completed a certain number of commands.
  • We want the saved copy to be in the same folder as the original file and have the same name, except instead of just the .esprit extension it will end with _Autosave.esprit
  • We want the auto save to reset whenever we open or start a new file.

To do this, we can use either the AfterCommandRun or BeforeCommandRun events of the Application to count the number of commands. We should also use the AfterDocumentSave event of the Document to update the auto save filename in case the user saves the file to a new name or new folder location.

Define a class for these event procedures:

C#
VB.NET
VBA
class AutoSaveHandler
{
private readonly Esprit.Application _application;
private int _commandCounter;
public int AutoSaveInterval;
public string AutoSaveName;
public AutoSaveHandler(Esprit.Application app)
{
_application = app;
AutoSaveInterval = 100;
_commandCounter = 0;
AutoSaveName = GetAutoSaveName();
_application.Document.AfterDocumentSave += AfterDocumentSave;
_application.AfterCommandRun += AfterCommandRun;
}
private void AfterCommandRun(int nCmdId, int nAuxId)
{
_commandCounter++;
if (_commandCounter >= AutoSaveInterval)
{
_application.Document.SaveCopyAs(AutoSaveName);
_commandCounter = 0;
}
}
private void AfterDocumentSave()
{
if (_commandCounter < AutoSaveInterval)
{
_commandCounter = 0;
AutoSaveName = GetAutoSaveName();
}
}
public string GetAutoSaveName()
{
var path = (_application.Document.FileName != string.Empty)
? Path.Combine(Path.GetDirectoryName(_application.Document.FileName), Path.GetFileNameWithoutExtension(_application.Document.FileName) + "_AutoSave.esprit")
: _application.TempDir + "AutoSave.esprit";
return path;
}
}
Public Class AutoSaveHandler
Private ReadOnly _application As Esprit.Application
Private _commandCounter As Integer
Public AutoSaveInterval As Integer
Public AutoSaveName As String
Public Sub New(app As Esprit.Application)
_application = app
AutoSaveInterval = 100
_commandCounter = 0
AutoSaveName = GetAutoSaveName()
AddHandler _application.Document.AfterDocumentSave, AddressOf Me.AfterDocumentSave
AddHandler _application.AfterCommandRun, AddressOf Me.AfterCommandRun
End Sub
Private Sub AfterCommandRun(nCmdId As Integer, nAuxId As Integer)
_commandCounter += 1
If _commandCounter >= AutoSaveInterval Then
_application.Document.SaveCopyAs(AutoSaveName)
_commandCounter = 0
End If
End Sub
Private Sub AfterDocumentSave()
If _commandCounter < AutoSaveInterval Then
_commandCounter = 0
AutoSaveName = GetAutoSaveName()
End If
End Sub
Public Function GetAutoSaveName() As String
Dim filePath As String = String.Empty
If _application.Document.FileName <> String.Empty Then
filePath = Path.Combine(Path.GetDirectoryName(_application.Document.FileName), Path.GetFileNameWithoutExtension(_application.Document.FileName) + "_AutoSave.esprit")
Else
filePath = _application.TempDir + "AutoSave.esprit"
End If
Return filePath
End Function
End Class
Public AutoSaveInterval As Long
Dim WithEvents MyApp As Esprit.Application
Attribute MyApp.VB_VarHelpID = -1
Dim WithEvents MyDoc As Esprit.Document
Attribute MyDoc.VB_VarHelpID = -1
Private CommandCounter As Long
Private AutoSaveName As String
Private IgnoreSave As Boolean
Private Sub Class_Initialize()
Set MyApp = Application
Set MyDoc = MyApp.Document
' call the GetAutoSaveName to get the AutoSaveName
AutoSaveInterval = 100
AutoSaveName = GetAutoSaveName
CommandCounter = 0
End Sub
Private Sub MyApp_AfterCommandRun(ByVal nCmdId As Long, ByVal nAuxId As Long)
' count the number of commands since last save, if over limit, then autosave
CommandCounter = CommandCounter + 1
If CommandCounter >= AutoSaveInterval Then
' because .SaveCopyAs will also trigger AfterDocumentSave event
IgnoreSave = True
Call MyDoc.SaveCopyAs(AutoSaveName)
IgnoreSave = False
CommandCounter = 0
End If
End Sub
Private Sub MyDoc_AfterDocumentSave()
' reset the values if necessary
If Not IgnoreSave Then
AutoSaveName = GetAutoSaveName
CommandCounter = 0
End If
End Sub
Function GetAutoSaveName() As String
' check to see if the filename is valid
If Len(Document.FileName) > 0 Then
' filename is valid, cut the .esp extension to form autosave filename
GetAutoSaveName = Left$(MyDoc.FileName, Len(MyDoc.FileName) - 4) & "_AutoSave.esprit"
Else
' temporary file, has not been saved yet
GetAutoSaveName = MyApp.TempDir & "AutoSave.esprit"
End If
End Function

Now we can use this class in a standard code module to launch an object of our auto save class each time we open or start a new document.

VB.NET
VBA
private AutoSaveHandler _testObject;
public override void Execute()
{
_testObject = new AutoSaveHandler(_application)
{
AutoSaveInterval = 10
};
}
Private _testObject As AutoSaveHandler
Public Overrides Sub Execute()
_testObject = New AutoSaveHandler(_application) With
{
.AutoSaveInterval = 10
}
End Sub
Dim AutoSaveTest As AutoSaveHandler
Sub AutoOpen()
Set AutoSaveTest = New AutoSaveHandler
' adjust the interval as desired, this will autosave every 10 commands
AutoSaveTest.AutoSaveInterval = 10
End Sub
Sub AutoClose()
Set AutoSaveTest = Nothing
End Sub

Open a file and then click a command 10 times, like Fit. After the tenth time, you should see the processing bar at the bottom briefly come on to indicate that the file is being saved. Pick the File Open command. Do you see the _AutoSave.esprit file listed below the original?

Click New to start a new file, and try other commands, like creating geometry. Does the auto save wait until you have created 10 new geometry elements before it saves, or does it save sooner? It may save sooner because after each element is created, the graphics are redrawn, and that redraw counts as a command, too. When you start working on a real file, 10 commands will probably be too small of an interval in which to save. You can easily increase the interval as you get more experience with auto save.
