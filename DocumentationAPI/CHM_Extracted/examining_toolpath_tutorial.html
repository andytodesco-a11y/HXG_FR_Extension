Examining the Toolpath

Now that we have seen how to create almost all 2D mill and lathe operations, the question might arise, just what comprises the operation? Is there a way to access the toolpath that is shown in the graphics for an operation? Well, if you look at the Operation object, you will see that it contains a ToolPath object. Although it is technically not a Collection, the ToolPath can be thought of as a collection of the items comprising the toolpath. Like other collections and collection-like objects (such as FeatureChain objects), it has a Count. Instead of an Item method however, which would allow you to modify the item, it has an ItemCopy method. This allows you to create a copy that you can interrogate, essentially making the original item read-only.

ItemCopy returns a ToolPathItem structure. The ToolPathItem contains an AuxCode, a Type (espToolPathItemType enumeration), and Data. The Data is the key. Depending on the AuxCode and Type, the data might be the technology, a graphical item, or some other information. We will examine the Data in more detail later. For now, let us just do some basic scanning of the ToolPath:

private void ScanToolPath()
{
_application.EventWindow.Clear();
_application.EventWindow.Visible = true;
foreach (Esprit.PartOperation operation in Document.PartOperations)
{
for (var i = 1; i <= operation.ToolPath.Count; i++)
{
var toolPathItem = operation.ToolPath.ItemCopy(i);
_application.EventWindow.AddMessage(EspritConstants.espMessageType.espMessageTypeInformation, "ExaminingToolpathTutorial", $"Operation {operation.Key} ({operation.Name}) Item {i} is type {toolPathItem.Type} (AuxCode = {toolPathItem.AuxCode})");
}
}
}
Public Sub ScanToolPath()
_application.EventWindow.Clear()
_application.EventWindow.Visible = True
For Each operation As Esprit.PartOperation In Document.PartOperations
For i As Integer = 1 To operation.ToolPath.Count
Dim toolPathItem = operation.ToolPath.ItemCopy(i)
_application.EventWindow.AddMessage(EspritConstants.espMessageType.espMessageTypeInformation, "ExaminingToolpathTutorial", $"Operation {operation.Key} ({operation.Name}) Item {i} is type {toolPathItem.Type} (AuxCode = {toolPathItem.AuxCode})")
Next
Next
End Sub
Sub ScanToolPath()
Application.EventWindow.Clear
Application.EventWindow.Visible = True
' loop through all operations
Dim Op As Esprit.PartOperation
For Each Op In Document.PartOperations
' loop through all items in the toolpath
Dim I As Long
For I = 1 To Op.ToolPath.Count
' notice how the ItemCopy method is used to return a ToolPathItem
' because ToolPathItem is a structure, not an object, the Set keyword is not used
Dim TPI As Esprit.ToolPathItem
TPI = Op.ToolPath.ItemCopy(I)
With Application.EventWindow
Call .AddMessage(EspritConstants.espMessageType.espMessageTypeInformation, "ExaminingToolpathTutorial", "Operation " & Op.Key & " (" & Op.Name & ") Item " & I & " is type " & TPI.Type & " (AuxCode = " & TPI.AuxCode & ")")
End With
Next
Next
End Sub
