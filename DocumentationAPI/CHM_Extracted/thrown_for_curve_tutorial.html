' have the user pick the arc or circle

TypeMask(1) = espArc
TypeMask(2) = espCircle

On Error GoTo 0

If E Is Nothing Then Exit Sub
Call DefaultAttributes(ConvertEllipseToCurve(E))
Else
' loop through the group, backwards since the function removes stuff

You can add a helix using the AddHelix method. The AxisPoint1 and AxisPoint2 arguments define the center axis of the helix. The curve then starts at the StartPoint and advances by the Lead value along the center axis with each full revolution.

Suppose that you do not like the values that the Helix Curve command prompts for, or the arguments that the AddHelix method asks for, for that matter. No matter combination of input you have for your helix, a code can be written to accommodate that information.

Sub ConstantRadiusHelixFromCenterAxisSegment()
' prompt the user for the desired helix input
Dim S As Esprit.Segment, Revolutions As Double, Radius As Double
On Error Resume Next
Set S = Document.GetSegment("Select Center Axis Segment")
On Error GoTo 0
If S Is Nothing Then Exit Sub
Revolutions = Val(InputBox("Enter Number of Revolutions", "Revolutions"))
If Revolutions = 0 Then Exit Sub
Radius = Val(InputBox("Enter Radius", "Radius"))
If Radius = 0 Then Exit Sub
' set the necessary parameters
Dim P(1 To 2) As Esprit.Point, SP As Esprit.Point, Lead As Double
Set P(1) = S.Extremity(espExtremityStart)
Set P(2) = S.Extremity(espExtremityEnd)
Set SP = Document.GetPoint(P(1).X + Radius, P(1).Y, P(1).Z)
Lead = S.Length / Revolutions
' create the curve
Call DefaultAttributes(Document.Curves.AddHelix(P(1), P(2), SP, Lead, 0))
End Sub
